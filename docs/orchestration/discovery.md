# Discovery Artifact

## Idea Summary

長文入力（数百〜1000文字程度）をLLMが5-7-5の上の句へ圧縮し、公開されるのは変換後の句のみとする。  
他ユーザーは、人間入力による7-7の下の句だけで返信する。返信部分はLLMを介さず、厳密な音数バリデーションで成立させる短歌SNS／感情圧縮インターフェース。

この企画の中心は、普通の短歌投稿サービスではなく、長文の内面をAIフィルターで詩的に変換し、「AIに解釈された自分」が流通する構造を試すことにある。

## Trigger Context

このアイデアは、「SNSはAIのフィルターがあれば平和な世界になる」という仮説から生まれた。

ユーザーは、SNSにおいて生の文章や感情がそのまま流通するのではなく、AIによる変換・フィルターを前提にした場があれば、関係性や表現の質が変わるのではないかと考えている。

## Tentative Hypotheses

- 短連歌／連歌に近い文化的構造を、AIによる上の句生成とSNS的返信に再解釈する企画と見なせる可能性がある。
- AIが投稿者の長文の内面を5-7-5に圧縮することで、投稿者本人の感情や人格が直接ではなく、詩的・抽象的に流通する可能性がある。
- 下の句を人間入力の7-7に制限することで、自由文返信よりも攻撃性や雑な反応が抑えられ、創作的な応答が促される可能性がある。
- 一方で、短歌としての品質や納得感が弱い場合、サービスの中心価値が成立しない可能性がある。
- フォロー機能などSNSらしい機能は、必要性と実装コスト次第で後から判断するのがよさそうである。

## Existing Alternatives

比較対象として近いもの:

- AIフィルター付きSNS
- 投稿内容が変換・翻訳・フィルタされてから流通するSNS
- 感覚的には、Xの自動翻訳に近いもの

比較対象として遠いもの:

- 短歌投稿サービス
- AI短歌生成サービス
- 通常の創作投稿サイト

文化的な参考としては、上の句と下の句を別の人が詠み合わせる連歌・短連歌に近い構造がある。ただし、この企画では上の句がAIによる内面圧縮で、下の句が人間による応答である点が異なる。

## Dissatisfaction with Alternatives

既存サービスに対して、明確な「足りなさ」や不満があるわけではない。

現在の認識では、そもそも「変換・フィルター前提のSNS」を知らないため、既存代替への不満というより、未探索の構造への関心が中心である。

## Strongest Pain

最も重要なのは、既存の明確な痛みではなく、次の構造仮説を検証することである。

AIフィルターによって投稿者の人格・感情を直接公開せずに流通させると、SNS上の関係性や表現は平和／創作的になるのか。さらに、上の句はAI、下の句は人間という分離によって、AI生成と人間同士の応答性を両立できるのか。
